# 合并优先级选择按钮与完成按钮

**日期**: 2026-06-11

## 背景

当前每条待办项的操作区包含两个相邻控件：优先级色点（`PriorityDot`）和复选框（`<input type="checkbox">`）。用户希望将两者合并为一个按钮，减少界面元素。

## 目标

用单个优先级点承担两个职责：
1. 点击打开选择器 → 可修改优先级
2. 选择器内提供完成/撤销操作 → 无需确认弹窗

## 交互设计

### 选择器布局

点击优先级点后弹出选择器，分两区：

```
┌──────────────────────────────────────────┐
│  [无]   [普通]  [重要]  [紧急]           │  ← 优先级行（现有）
│  ─────────────────────────────────────── │  ← 分隔线
│           [ ✓ 完成 ]                     │  ← 完成/撤销行
└──────────────────────────────────────────┘
```

- 已完成时，该行显示"↩ 撤销完成"
- 点击后直接切换状态，无确认弹窗

### 完成状态的点外观

优先级点保持原有颜色，叠加白色小对钩：

| 状态 | 外观 |
|------|------|
| 未完成（有优先级） | 原色实心圆 |
| 已完成（有优先级） | 原色实心圆 + 中心白色 ✓ |
| 未完成（无优先级） | 空心圆（带边框） |
| 已完成（无优先级） | 空心圆 + 中心 ✓（边框色） |

## 组件改动

### `PriorityDot.vue`

- **新增 prop**: `done: Boolean`
- **新增 emit**: `toggle:done`
- **样式**: `done === true` 时，点的伪元素显示 ✓
- **选择器**: 底部加分隔线 + 完成/撤销按钮，点击时 `emit('toggle:done')`

### `TodoList.vue`

- 移除 `<input type="checkbox">`
- 向 `<PriorityDot>` 传入 `:done="todo.status === 'done'"`
- 监听 `@toggle:done="store.toggleTodo(todo.id)"`
- 删除 `confirmToggle` 函数（完成动作不再需要确认）

### 不变的部分

- 删除确认弹窗逻辑
- 双击编辑功能
- 优先级设置的其余逻辑
- `store.toggleTodo` 实现

## 范围外

- 优先级色点在输入框（`TodoInput`）中的添加任务流程 — 不涉及
- 移动端手势支持 — 不涉及
